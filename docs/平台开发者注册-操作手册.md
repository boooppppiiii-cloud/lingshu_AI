# 平台开发者注册 + App 审核 操作手册（复制粘贴版）

> 目标：为「官方 API 直连 + 自写 OAuth + 自托管」准备开发者账号与 app 审核。
> 用法：每节按顺序操作；**登录 / 2FA / 接受协议 / 上传证件 / 最终提交** 由你本人完成，中间要填的内容直接抄本文。
>
> ⚠️ 先替换全文占位符：
> - `{{PROD_DOMAIN}}` = 你的生产域名（如 `app.lingshu.ai`）；开发期用 `localhost`
> - `{{COMPANY}}` = 公司法定名称（如 义乌XX商贸有限公司）
> - `{{APP_NAME}}` = 对外 app 名（建议 `灵枢 AI` / `Lingshu AI`）
> - `{{SUPPORT_EMAIL}}` = 支持邮箱
>
> 统一回调地址规范（OAuth callback）：
> `https://{{PROD_DOMAIN}}/api/overseas/oauth/<platform>/callback`
> 开发期再加：`http://localhost:8790/api/overseas/oauth/<platform>/callback`

## 1. Meta（一套覆盖 Facebook 主页 + Instagram Reels）｜性价比最高，先做这个
门户：https://developers.facebook.com → 用公司 FB 账号登录 → 注册成为开发者。

1. **创建 App**：类型选 **Business** → App 名 `{{APP_NAME}}` → 关联你的 Business Portfolio。
2. **加产品**：添加 **Facebook Login**、**Instagram**（Instagram API with Instagram Login / Graph API）。
3. **OAuth 回调**：Facebook Login → Settings → Valid OAuth Redirect URIs 填：
   `https://{{PROD_DOMAIN}}/api/overseas/oauth/meta/callback`
4. **申请权限（Permissions，需 Advanced Access 走 App Review）**：
   `pages_show_list`, `pages_read_engagement`, `pages_manage_posts`,
   `instagram_basic`, `instagram_content_publish`,
   `read_insights`（互动数据）, `business_management`
5. **业务验证 Business Verification**：上传 `{{COMPANY}}` 营业执照 + 公司信息（免费）。
6. **App Review 用途说明**（粘贴，见第五节模板）+ **录屏演示**（脚本见第七节）。

---

## 2. TikTok（Content Posting API）
门户：https://developers.tiktok.com → 注册开发者 → Verify your business（交公司信息）。

1. **创建 App** → 名称 `{{APP_NAME}}`。
2. **加产品**：**Login Kit** + **Content Posting API**。
3. **Scopes**：`user.info.basic`, `video.upload`, `video.publish`（如要拉数据再加 `video.list` 等）。
4. **Redirect URI**：`https://{{PROD_DOMAIN}}/api/overseas/oauth/tiktok/callback`
5. **审核**：提交 **演示录屏**（脚本见第七节，重点演示"用户主动授权 + 发布 + 可见性说明"）。
   注意：**未过审前，通过 API 发布的内容只能私密可见**，过审后才放开。

---

## 3. YouTube / Google
门户：https://console.cloud.google.com → 新建 Project。

1. **启用 API**：YouTube Data API v3、YouTube Analytics API。
2. **OAuth consent screen**：User Type 选 **External**；填 App 名 `{{APP_NAME}}`、support email、首页/隐私/条款 URL；添加你自己为 **Test users**（开发期）。
3. **Scopes**：`.../auth/youtube.upload`, `.../auth/youtube.readonly`, `.../auth/yt-analytics.readonly`。
4. **凭据**：创建 OAuth Client ID（Web application）→ Redirect URI：
   `https://{{PROD_DOMAIN}}/api/overseas/oauth/youtube/callback`
5. **验证**：上线给外部用户用需走 OAuth 验证（免费）；**确认所用 scope 是否落入 restricted**——youtube.upload/readonly 通常是 sensitive（免费审核），若被判 restricted 才触发第三方 CASA（$500–4500/年，多数情况用不到）。

## 4. Pinterest
门户：https://developers.pinterest.com → 注册 → 创建 App。

1. **App 名** `{{APP_NAME}}`，填用途说明。
2. **Scopes**：`user_accounts:read`, `pins:read`, `pins:write`, `boards:read`。
3. **Redirect URI**：`https://{{PROD_DOMAIN}}/api/overseas/oauth/pinterest/callback`
4. 申请从 Trial 升到标准访问（填用途，免费）。

## 6. 隐私政策 / 服务条款（最小模板，挂到 /privacy、/terms）
**Privacy Policy 要点**（自己补全公司信息）：
- 收集什么：账号基本信息、所连社媒账号的发布权限与公开互动数据。
- 用途：仅用于用户在本产品内主动发起的发布与数据分析。
- 存储与安全：access token 加密存储；用户可随时断开授权并删除。
- 第三方：列出 Meta/TikTok/Google/Pinterest/Shopify。
- 联系方式：`{{SUPPORT_EMAIL}}`。
- 删除数据入口与流程。
（正式上线建议让法务过一遍；先用模板满足审核要求即可。）

## 7. 演示录屏脚本（Meta / TikTok 审核用，2–3 分钟）
1. 打开 `{{APP_NAME}}`，进入「插件 / 社媒授权」。
2. 点「连接 Facebook / TikTok」→ 跳转官方授权页 → 用户**主动同意**所列权限。
3. 回到产品，显示**已连接的账号信息**（头像/名称/粉丝）。
4. 进入混剪工作台生成一条视频 → 在导出步点「一键发布」→ 选账号 → 发布成功。
5. 进入**数据面板**，展示该条内容的互动数据（播放/点赞/评论）。
6. 旁白说明：所有发布均由用户主动触发，权限最小化，可随时断开。
