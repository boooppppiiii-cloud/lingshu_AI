# 社媒插件做"真实可用"——集成选型调研

> 决策文档，给"插件做真"拍板用。当前结论：**先调研不动手**。
> 项目：`overseas-workbench`，跨境电商社媒发布/分析工具。

---

## 一、现状（为什么说"还不是真的"）

插件页（`src/components/PluginsPage.tsx` + `server/routes/plugins.ts`）：
- **真连**：实时汇率（调 exchangerate-api）、Shopify（`testShopify`）、AI 翻译（内置 LLM）。
- **占位**：TikTok / Instagram / Facebook / Pinterest / WhatsApp / Amazon / Google 翻译——能安装、能存配置、点「测试」只走 `default` 分支回"需配置 API Key"，**不真连对方平台**。
- **缺三块**：① OAuth 授权跳转（现在是手填 Token 表单）② 真实 test（验证 token + 拉账号信息）③ 连接后的**账号面板**（主页信息/粉丝/最近发布/一键发布）。
- 发布权限的硬约束：**TikTok/Meta/Pinterest 发布权限都要过各自平台的 app 审核**（Meta App Review、TikTok 审核最严），无论自建还是用聚合平台。

## 二、三条路线对比

| 维度 | A 全自建 | B OAuth 托管 (Nango/Composio) | C 统一聚合 API |
|---|---|---|---|
| 做法 | 自己接各平台 OAuth + 官方 API | 授权/刷新托管，API 自己调 | 一套 API 发所有平台 |
| 上手速度 | 慢 | 中 | **快** |
| 账号面板/数据 | 自己实现 | 自己实现 | **开箱即用** |
| 抽成/费用 | 无 | 订阅(OAuth 托管) | 按量/订阅 |
| 控制力&数据归属 | **最强** | 强（数据在自己手里） | 弱（依赖第三方） |
| 平台 app 审核 | 必须，全自己扛 | 必须 | 初期常可借它家已过审 app |
| 维护负担 | 高（token 刷新/各平台变更） | 低 | 低 |

### 候选产品
- **C 聚合**：Ayrshare（商用，覆盖广）；**Postiz**（开源可自托管，github gitroomhq/postiz-app）；**Mixpost**（开源可自托管）；Blotato / Late / upload-post（商用）。
- **B OAuth 托管**：Nango（nango.dev，OAuth-as-a-service，几百个 provider）；Composio。
- **数据/分析向**：Phyllo / InsightIQ（创作者数据）。

## 三、建议（针对早期、自托管后端、Free Plan 的跨境卖家）

1. **MVP 先走 C，且优先开源可自托管的 Postiz / Mixpost**——你本就自托管后端，这俩本地部署、无按条抽成，最快把"连接账号 + 账号面板 + 一键发布"跑通。
2. 量起来、要更深数据时，把高频平台迁到 **B（Nango 托管 OAuth + 自调官方 API）**，拿回数据与控制力。
3. **A 全自建**只在某平台成为核心、值得专门投入时再做。

落到现有代码的演进路径：
配置/测试模态 → 「连接」走 OAuth → 存 token → 「测试」真调 `/me` 拉账号名/头像/粉丝 → 新增**账号面板**（主页信息 + 最近发布 + 一键发布入口）→ 和导出步骤的"一键发布"打通。

## 四、需求已确认（2026-06）
- **平台优先级**：优先**视频平台**（灵枢的短视频生成最能赋能）→ TikTok、YouTube(Shorts)、Instagram Reels、Facebook Reels；Pinterest 视频 Pin 次之。
- **数据**：要拉**互动 + 转化**数据做分析。
- **费用**：见下方报价。
- **开发者账号**：**还没有**（未启动任何平台 app 审核）——这点直接影响选型，见第六节。

## 五、关键区分：互动 ≠ 转化（这决定架构）
- **互动数据**（播放/点赞/评论/分享/收藏/完播）：发布聚合器（Ayrshare 自带 analytics）或各平台 Insights API 都能给。
- **转化数据**（点击→加购→下单→ROAS）：**organic 发布 API 给不了**。来源是：
  1. 各平台**广告/转化 API + 像素/Events**（TikTok Events API、Meta Conversions API、Pinterest Tag）；
  2. **UTM 链接回挂 Shopify 订单**做真实归因——**你已经有 Shopify 插件**，这条几乎零外部成本、ROI 最高。
- 结论：**发布+互动**走聚合器；**转化**主要靠 UTM+Shopify（自建，便宜）+ 平台像素。别指望一个聚合器把转化也包了。

## 六、当前报价（2026-06 实时核对）

|---|---|---|---|
| **Ayrshare** | 托管聚合 API | Premium **$149/mo**（1 profile，analytics 含于 Premium+）；Business **$499–599/mo**（~30 profiles），超出 $2.49–8.99/profile | 可借它已过审 app 发布，**绕开初期审核** |
| **Postiz** | 开源，可自托管 | **自托管免费**；云版 $29 / $39 / $49 / $99 每月（5/10/30/100 渠道） | 自托管=用你自己的平台 app key |
| **Mixpost** | 开源自托管 | Pro **一次性 $299**（含 1 年更新，无限用户）；Enterprise $1,199（可搭 SaaS） | +服务器 ~$5–50/mo，零抽成 |
| **Nango** | OAuth 托管 | Free 10 连接；Starter **$50/mo**(20 连接+$1/个)；Growth **$500/mo**(100 连接)；企业版面议 | 只托管授权/刷新，API 自己调 |

## 七、给你的推荐（视频优先 + 要转化分析 + 暂无开发者账号）
**关键约束**：你没有开发者账号 → **自托管(Postiz/Mixpost)要你自己建各平台 app 并过审**，会立刻撞上审核墙；而**托管聚合(Ayrshare / Postiz 云版)能挂它家已过审 app，先发起来**。

- **Phase 1（最快见真）**：用**托管聚合**先把"连接视频账号 + 发布 + 互动数据"跑通——重分析就 **Ayrshare Premium $149/mo**（analytics 含），重省钱就 **Postiz 云版 $29–49/mo**。同时**自建 UTM→Shopify 转化归因**（你已有 Shopify，外部成本≈0）。
- **Phase 2**：互动数据要更深/跨平台受众画像时，加 **Phyllo**（面议）或上 Ayrshare Business。
- **Phase 3**：发布量大、要去抽成/全控时，迁到 **Nango($50–500/mo)+官方 API**，并自己过 TikTok/Meta app 审核。

> 注：TikTok/Meta 发布的高级权限，最终都要过对方 app 审核；聚合器的价值就是**初期借它已过审 app 先发**，给你争取做审核的时间。

## 八、多租户成本：每客户 vs 平台级（重要）
你是多租户 SaaS（很多客户，每客户连自己的号）。**大部分聚合器/Nango 按"连接的社媒账号数"计费 → 一个平台账号能技术上服务所有客户，但成本随客户数线性涨。** 只有自托管 / 官方 API 是平台级固定成本、与客户数无关。

| 方案 | 一账号服务所有客户 | 成本随客户数涨 |
|---|---|---|
| Ayrshare | ✅（每客户建一个 User Profile） | ⚠️ 按 profile 计费 |
| Nango | ✅ | ⚠️ 按 connection 计费 |
| Phyllo | ✅ | ⚠️ 按 profile 计费 |
| Postiz / Mixpost 自托管 | ✅ | ✅ 不涨 |
| 平台官方 API 自建 | ✅（客户 OAuth 进你的 app） | ✅ 不涨 |

⚠️ 更正：Ayrshare **Premium $149 是单 profile（一家公司用），不够多租户**；多租户要 **Business 版**。
按 Business 估算（每客户≈1 profile）：100 客户 ≈ **$950/月**；1,000 客户 ≈ **$5,400/月**——会成为最大可变成本。
Nango（仅托管 OAuth，便宜些）：100 客户×2 号 ≈ **$600/月**；1,000 客户×2 号 ≈ **$2,400/月**。
自托管：100 还是 10,000 客户**一个价**（Mixpost 一次性 $299 + 服务器），零按客户抽成。

## 九、更省钱方案（用户嫌贵后定的方向）：官方 API 直连 + 自写 OAuth + 自托管
**绕开一切按客户抽成，只剩服务器成本。** 代价从"每月付费"换成"一次性开发 + 平台审核"。
1. 每平台建**一个你自己的 app**，所有客户 OAuth 进来，**API 调用免费**。
2. **OAuth 不用 Nango**——用开源库（simple-oauth2 等）自写授权+刷新，省掉每连接 $1。
3. 视频平台只做最值的三个：**Meta（FB+IG Reels 一套 token）、TikTok、YouTube**。
4. 转化继续 **UTM→Shopify**（已有 Shopify，外部成本≈0）。
5. 代价：每平台**过 app 审核**（免费，见第十节）+ 自己运维。

**推荐先做 Meta 样板**（一套 OAuth 同覆盖 FB+IG Reels），跑通 `连接→拉账号/互动数据→一键发布`，再复制 TikTok、YouTube。
（商业兜底：若仍想用聚合器，可**按渠道数向客户收费**把抽成转嫁，并对 profile 懒创建/断开即删压成本。）

## 十、平台审核费用：基本为 0（2026-06 核对）
| 平台 | 开发者账号/API | 审核 | 费用 |
|---|---|---|---|
| Meta（FB+IG） | 免费 | App Review + 业务验证（传文件，不收费） | **¥0** |
| TikTok | 免费，无按调用费/订阅 | Content Posting API 审核（交演示录屏；过审前内容仅私密可见） | **¥0** |
| YouTube/Google | 免费（有免费配额） | OAuth 验证免费；提配额走免费合规审计 | **¥0**（见例外） |
| Pinterest | 免费 | 免费 | **¥0** |

- **唯一可能花钱**：Google **受限范围(restricted scope)** 的第三方安全评估 CASA，**$500–$4,500/年**，Google 自己不收、是第三方评估机构收，且只有用到受限范围才触发；视频发布通常是 sensitive scope，多半免费。
- Meta Verified（蓝标订阅）与 API 接入**无关**，别买。
- **真正成本是时间/材料**，不是钱：见第十一节。

## 十一、提交审核 材料清单（现在就能去排队）
通用：
- [ ] 一个**公开可访问的隐私政策页 + 服务条款页**（自己写即可，模板见网上）。
- [ ] App 用途说明 / 数据如何使用。
- [ ] 一个可访问的产品网站/域名（已有）。

分平台：
- [ ] **Meta**：业务验证文件（营业执照即可）；App Review 录屏演示"用户授权→发布 Reels→读取互动数据"的完整流程。
- [ ] **TikTok**：Content Posting API 审核的**演示录屏**（重点是合规：用户主动授权、可见性说明）。
- [ ] **YouTube/Google**：OAuth consent 配置 + 用途说明；如提配额，做免费合规审计；确认所用 scope 是否落在 restricted（多半否）。
- [ ] **Pinterest**：开发者账号 + app 用途说明。

> 结论：**审核不花钱，花的是几天填材料 + 等审核周期（数天到数周）。** 越早注册开发者账号、挂上隐私政策、提交审核越好，与后续选型不冲突。

## 十二、竞品功能页参考（账号面板 / 分析 / 发布 UI 借鉴，2026-06 调研）
不要凭空设计账号面板，按这些真实竞品的功能页结构来：

| 竞品 | 定位 | 借鉴点 |
|---|---|---|
| **Creatify** | 商品 URL→AI 视频广告（≈混剪工作台） | 一站式工作流；Dashboard 浏览广告库(含竞品在投)、逐条预览/下载/编辑/看表现迭代；多语言重渲染；模板复用 |
| **Predis.ai** | AI 内容+多平台发布+分析 | 分析仪表盘(互动率/曝光/受众画像)；**AI 优化建议**；竞品分析；单面板发多平台 |
| **Metricool** | 社媒分析+排期（分析最强） | 顶部**账号切换下拉**；分析分 **Community(涨粉/受众画像)** + **Posts(reach/赞/评/收藏/分享/互动/点击)**；专门**视频指标(观看/观看时长/完播率)**；自定义报表 |
| **Later** | 排期+分析+Link in Bio | 连接后顶部 profile 下拉；曝光/触达/**网站点击**/涨粉/**最佳表现帖**；**Link in Bio + UTM** 追踪转化(=我们 UTM→Shopify 的真实对应) |

**借鉴版"账号面板"结构（全部有竞品出处）**：
顶部**账号切换下拉** → **受众**(涨粉 + 年龄/性别/地区画像) → **内容**(reach/赞/评/收藏/分享/点击 + **最佳表现帖**) → **视频**(观看时长/完播率) → **转化**(Link in Bio + UTM → Shopify 下单/销售额) → **AI 优化建议**。

修正/对应：
- "平台 chips 一排" → 改成**账号切换下拉**(主流做法)。
- 分析按 **受众/内容/视频** 三区(Metricool 真实分法)，看"最佳表现帖"而非"最近发布"。
- 转化 = **Link in Bio + UTM**(Later 实锤) + Shopify。
- 加 **AI 优化建议**面板(Predis 差异点)。
- **竞品/灵感广告库** → 我们已有「灵感大屏」；**排期日历** → 已有「定时任务」，不重做。
