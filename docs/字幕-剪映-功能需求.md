# 灵枢 × 本地剪映：字幕能力 功能背景与详细需求

> 给「另开的桌面端对话」用的自包含交接文档。读完即可直接动手，无需回看上一段对话。
> 项目：`overseas-workbench`（义乌跨境 · 海外社媒混剪工作台）。前端 5174 / 后端 8790 / 桌面 Electron。

---

## 一、背景

混剪工作台（`src/components/AiCreateStudio.tsx`）的流程是：选模式 → 选素材 → 口播脚本(AI+TTS) → 配乐 → 封面 → **成片预览** → 导出/发布。

成片在**桌面客户端**用本机原生 ffmpeg 真实合成（`desktop/render.cjs`）；纯网页端只能模拟进度。

需求：**成片要按口播内容自动配字幕**。已和用户确认方向：
- **对齐方式 = 真 ASR**（对生成的 TTS 配音跑 Whisper 逐词时间戳），不用脚本时间标记/字数比例那种粗对齐（那个只是网页端兜底）。
- **语言 = 开关切换**：目标语言字幕 ↔ 目标语言 + 中文译文（双语）。
- **样式 = 默认沿用封面样式体系**（字体/颜色/粗细），我们只做"够用的默认字幕"；逐字动效、卡点这类精修**交给本地剪映**（一键跳转）。

分两层落地：**A 层（网页可见可验证）已完成**；**B 层（桌面专属）就是本文档要做的**。

## 二、A 层已经做好的（前端 + 共享类型 + 服务端，已合入）

这些已经在代码里，B 层直接对接即可，**不要重复造**：

1. **共享类型** `src/lib/studioApi.ts`
   - `SubCue { start:number; end:number; text:string; zh?:string }`（start/end = 相对成片起点的秒）
   - `SubtitleSpec { mode:'off'|'target'|'bilingual'; cues:SubCue[]; style:Partial<CoverStyle> }`
   - `RenderSpec.subtitles?: SubtitleSpec`、`RenderManifest.subtitles?: SubtitleSpec`
   - `DesktopRenderBridge`（`window.desktopRender`）—— **B 层要往这里加新方法**。
   - `studioApi.translate({text,target})` 翻译接口（双语字幕的中文译文就靠它，已上线）。

2. **前端** `src/components/AiCreateStudio.tsx`
   - `buildCues(script, dur)`：A 层兜底对齐（脚本 `[Hook·0-3s]` 标记优先，否则按 TTS 时长/字数比例）。**B 层有 ASR 后，用 ASR 结果替换 cues 来源即可，下游 UI/manifest 不用动。**
   - 状态：`subtitlesOn`、`subMode`（target/bilingual）、`cues`(memo)、`cueZh`(双语译文)、`subStyle`(封面样式派生：底部居中)。
   - 成片预览页已有：**字幕卡片**（开关 + 目标语言/双语切换 + 逐句 cue 列表）、播放器上的**字幕叠层**、**「用剪映精修」按钮**（`openInCapcut()`）。
   - `goPreview()` 已把 `subtitles` 塞进 render spec。
   - `openInCapcut()` 现状：`getDesktopRender().openInCapcut?.(...)` 有桥就调，没桥就提示——**B 层要在桥上实现 `openInCapcut`**。

3. **服务端** `server/routes/studio.ts`
   - `buildManifest()` 已把 `spec.subtitles`（mode!=='off' 时）透传进 manifest。

4. **桌面端现状**
   - `desktop/main.cjs`：`ipcMain.handle('render:start', ...)` → `composite()`（render.cjs），完成后 `shell.showItemInFolder`。
   - `desktop/preload.cjs`：`contextBridge.exposeInMainWorld('desktopRender', { available, render, onProgress })`。
   - `desktop/render.cjs`：ffmpeg 合成。已有 `findFont()`（CJK 字体探测）、`drawtext()` 助手；**当前只烧了封面标题(y=h*0.10) 和 firstHook(y=h*0.80) 两条静态 drawtext**。字幕烧录就在这里扩展。

## 三、B 层详细需求（本次要做）

### 需求 1：ASR 逐词/逐句时间戳（替换兜底对齐）
- 输入：TTS 配音音频（manifest.voiceover.url，本机已下载到临时目录）。
- 用 whisper.cpp（带一个 small/base 模型）或等价本地 ASR，输出逐句（最好逐词）时间戳。
- 主进程加 IPC `asr:align`（入参音频路径/URL，出参 `SubCue[]`），`preload.cjs` 暴露成 `window.desktopRender.alignSubtitles(audioUrl) => Promise<SubCue[]>`。
- 前端：检测到桥有 `alignSubtitles` 时，进预览/生成配音后用它的结果覆盖 `cues`（结构已统一，替换数据源即可）；无桥时保持 `buildCues` 兜底。
- 双语：ASR 出目标语言 cue 后，中文译文仍走 `studioApi.translate`（已有），或桌面端本地翻译，二选一。

### 需求 2：字幕烧录进 MP4（`render.cjs`）
- manifest.subtitles 存在且 mode!=='off' 时，按 cue 烧字幕。
- **推荐生成 ASS 字幕**（比逐条 drawtext 好控样式/双语/位置），用 ffmpeg `subtitles=xxx.ass`（或 `ass=`）滤镜挂到视频链路 `[vout]` 上。
  - 样式映射：`subtitles.style` 的 font/color/weight → ASS Style（注意 ASS 颜色是 `&HBBGGRR`）；底部居中、白字黑描边(Outline)、避开底部安全区。
  - 双语：目标语言主字幕 + 中文副字幕（小一号），可用两个 ASS Style 或同一 cue 两行。
  - 字体沿用 `findFont()`（CJK 安全），自定义导入字体（coverStyle.fontFamily）按需处理。
- 兜底：找不到字体/ASS 失败时退化为不烧字（别让整条合成失败）。

### 需求 3：一键跳转本地剪映（CapCut/剪映 草稿）
- 主进程加 IPC `capcut:open`，`preload.cjs` 暴露成 `window.desktopRender.openInCapcut(payload)`（前端调用点已就绪）。
- 目标：把素材轨 + 字幕轨（cues）+ 基本时间线生成**剪映草稿**，写入剪映本地草稿目录，并唤起剪映 App，让用户做逐字动效/卡点精修。
- ⚠️ **最大风险**：剪映/CapCut 草稿格式（`draft_content.json` 等）**未公开、随版本变**。建议：先做**最小可行草稿**（一条素材轨 + 一条字幕轨），在用户本机那一版剪映上试打开成功，再逐步加轨。先在 mac 剪映国内版上试。
- 草稿目录因平台/版本而异，需可配置 + 自动探测；唤起用 `shell.openPath` 或 deep link。

### 需求 4：开发联调工作流（务必先验证可行）
```bash
npm run dev:server                                  # 后端 8790
npm run dev                                          # 前端 5174（热更新）
DESKTOP_URL=http://localhost:5174 npm run desktop    # Electron 加载热更新前端 + 注入桥
```
- 改前端 React → 热更新即时看；改 main.cjs/preload.cjs/render.cjs → 重启那条 electron 命令。
- 现仓库**没配 electron-builder**（无 `build` 字段），开发期不需要安装包；要发 .dmg/.exe 是后续一次性配置。

---

## 四、建议推进顺序
1. 先打通**开发联调工作流**（确认 electron 能加载 5174 并注入桥）。
2. **需求 2（ASS 烧录）** 先做——用 A 层已有的 cues 就能立刻在桌面端看到真字幕成片，价值最快兑现。
3. 再做**需求 1（ASR）** 把对齐换成精确的。
4. 最后啃**需求 3（剪映草稿）**——风险最高，先最小验证。

## 五、验收
- 桌面端合成出的 MP4 里，字幕按口播内容出现、时间对得上、样式像封面体系、双语模式下中文在下方。
- 关字幕则不烧字。
- 「用剪映精修」能在本机打开剪映并带入素材+字幕草稿。
